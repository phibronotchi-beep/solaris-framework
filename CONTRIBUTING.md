# Contributing to Phyllux Framework

Thank you for your interest in contributing to the Phyllux Framework.

## Repository Role

This repository defines ethical, licensing, and collaboration structures for independent inventors. It provides templates, guides, and a framework for protecting intellectual property.

**Boundaries:**
- This repository does NOT contain invention claims
- This repository does NOT contain embodiment descriptions
- This repository focuses on governance, licensing, and coordination

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest improvements
- Be clear and specific
- Include relevant file paths and line numbers if applicable

### Improving Documentation

- Fix typos or clarify language
- Add examples where helpful
- Improve cross-linking between documents
- Ensure consistency with repository role

### Enhancing Templates

- Make templates clearer and more usable
- Add helpful comments or instructions
- Ensure legal language remains appropriate
- Test templates for clarity

### Adding Guides

- Share your experience using the framework
- Write step-by-step guides
- Document common pitfalls and solutions
- Keep language neutral and technical

## Guidelines

### Language and Tone

- Use neutral, technical language
- Avoid hype or promotional language
- Preserve the calm, authoritative tone
- Focus on clarity and precision

### Content Boundaries

- Do NOT add invention claims
- Do NOT add embodiment descriptions
- Do NOT modify core licensing model without discussion
- Do NOT weaken disclaimers or safety language

### Code Contributions

- Scripts should be well-documented
- Follow existing code style
- Include appropriate safety checks
- Test before submitting

## Process

1. Open an issue to discuss significant changes
2. Fork the repository
3. Create a branch for your changes
4. Make your changes
5. Submit a pull request with a clear description
